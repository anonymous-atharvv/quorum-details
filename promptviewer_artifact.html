Identity & Mission

Foundation

QUORUM — Collective Intelligence Platform. A B2B AI platform that makes organizational group decisions measurably better by combining pre-meeting anonymous elicitation, real-time meeting intelligence, and longitudinal outcome tracking with compounding intelligence.

🔴 Core Insight Every existing meeting tool (Otter.ai, Fireflies, Slido) solves efficiency. Quorum solves decision quality — a fundamentally different and more valuable problem. Organizations lose $37B annually in the US to decisions made poorly in meetings.
🎯
HiPPO Effect
Highest-paid person's opinion dominates and suppresses all other voices
🐑
Groupthink
Social pressure kills honest dissent before it's voiced in the room
🔇
Info Pooling Failure
People have the answer but never say it due to hierarchy or fear
⚓
Anchoring Bias
First speaker frames the entire conversation direction permanently
⚠️ What Quorum Is NOT Not a transcription tool. Not a task manager. Not a performance evaluation tool — NEVER used to assess individuals. Not surveillance — all individual input is architecturally anonymized at the point of collection.

Anonymization System

Cryptographic
🔴 Cryptographic Guarantee Even with full database read access + the complete user list, an attacker cannot de-anonymize any respondent without the HMAC secret stored in AWS Secrets Manager. This is the core privacy guarantee of the entire product.
python — app/core/security.py
import hashlib, hmac
from app.core.config import settings

def anonymize_respondent(user_id: str, meeting_id: str) -> str:
    """
    One-way HMAC-SHA256. Properties:
    - Deterministic: same user+meeting → same hash
    - Irreversible: cannot recover user_id from hash
    - Meeting-scoped: different hash per meeting (no cross-meeting tracking)
    - Secret-protected: requires RESPONDENT_HASH_SECRET

    Secret is in AWS Secrets Manager ONLY. Never in DB or code.
    Rotate annually. Never rotate while survey_open meetings exist.
    """
    secret = settings.RESPONDENT_HASH_SECRET.encode()
    message = f"{user_id}:{meeting_id}".encode()
    return hmac.new(secret, message, hashlib.sha256).hexdigest()[:20]

def anonymize_speaker(platform_id: str, meeting_id: str) -> str:
    """Meeting-scoped. Same person = different hash per meeting."""
    secret = settings.SPEAKER_HASH_SECRET.encode()
    message = f"{platform_id}:{meeting_id}".encode()
    return hmac.new(secret, message, hashlib.sha256).hexdigest()[:16]

def verify_survey_token(token: str, db_token: str | None) -> bool:
    """Constant-time comparison — prevents timing attacks."""
    if not db_token: return False
    return hmac.compare_digest(token.encode(), db_token.encode())

Required adversarial test coverage (tests/unit/test_anonymization.py):

✓Same user + same meeting → same hash (determinism for duplicate-check)
✓Same user + different meetings → different hashes (no cross-meeting tracking)
✓Different users + same meeting → different hashes (no confusion)
✓Hash length fixed regardless of input length (no length leakage)
✓Brute force with full user list fails without secret (attacker simulation)
✓user_id never appears in application logs (no accidental PII)
✓Two hashes from different meetings share no detectable pattern (no correlation)

async def generate_tension_map(meeting_id) -> tuple[TensionMapOutput | None, str]:
    rate = await get_response_rate(meeting_id)

    if rate < 0.30:
        return None, "insufficient_responses"  # hard refuse

    output = await run_tension_analyst(meeting_id)

    if rate < 0.50:
        output.confidence = min(output.confidence, 0.40)
        output.confidence_caveat = (

        return output, "low_confidence"

# TIER 1: HiPPO (rule-based, ~1ms) — runs EVERY 30s
def detect_hippo(speaking, total_sec, elapsed_sec, alerts_delivered):
    if elapsed_sec < 300: return None  # 5-min minimum
    if alerts_delivered >= 2: return None  # cap at 2 per meeting
    for hash, secs in speaking.items():
        pct = secs / total_sec
        if pct > 0.45: urgency = "high"
        elif pct > 0.38: urgency = "medium"
        else: continue
        return Alert(type="hippo", urgency=urgency, ...)

# TIER 2: Groupthink Precheck (rule-based, ~1ms)
consensus_signals = ["i agree", "totally", "let's go with",
    "sounds good", "we're aligned", "move forward", "same page"]
# fires LLM if count ≥ 3 AND tension_score > 0.35 AND elapsed > 8min

# TIER 3: Missing Perspective (Pinecone semantic, ~200ms)
# cosine_similarity(embed(last 40 chunks), embed(tension_topic)) < 0.35
# → topic not discussed → fire LLM
# cap: 3 missing-perspective alerts max per meeting

# LLM EVALUATION (only if tier 2 or 3 fires):
# Returns: {"action": "monitor"} (most common case)
# OR {"action": "intervene", "type": ..., "message": ..., "suggested_question": ...}
# Rule: intervene ONLY when >80% confident. False positives destroy trust.
4
Post-Mortem Generator
Trigger: Meeting ends (≤24hr)
Latency: <60s
Key rule: Assumptions must be SPECIFIC & MEASURABLE
Generates structured post-mortem with executive summary, decisions (options considered, key assumptions, measurable success criteria at 30/90/180 days), process observations (warning signs detected, what went well), and open questions. Dissenting views come from anonymous pre-survey ONLY. Never attributed.

Pattern Detector
Trigger: Weekly, Sunday 03:00 UTC
Latency: <5 min
Orchestration: LangGraph multi-agent
Min sample: 5 data points per pattern
Pre-computes domain accuracy (Wilson CIs), overconfidence delta, temporal patterns, participation correlation — then passes to LLM for interpretation. Only surfaces patterns with statistical significance (n≥5). Actionable interventions only. No patterns that could identify individuals.

architecture — real-time flow
Meeting audio (Zoom/Teams/Meet)
       │
       ▼ Deepgram Nova-2 WebSocket (~300ms latency)
       │ interim_results=False, punctuate=True, diarize=True
       │
       ├──► anonymize_speaker(zoom_id, meeting_id) → speaker_hash
       │
       ├──► InfluxDB: speaking_time {org_id, meeting_id, speaker_hash, seconds}
       │
       ├──► Redis: rolling_buffer (last 200 chunks ≈ 10min, TTL 4hr)
       │
       ├──► PostgreSQL: live_transcript_chunks (persisted)
       │
       └──► WebSocket broadcast: transcript_chunk to facilitator

EVERY 30 SECONDS (asyncio task):
       │
       ├── Tier 1 HiPPO (O(n) speakers, ~1ms)
       │
       ├── Tier 2 Groupthink precheck (string match, ~1ms)
       │         if fires → Tier 3
       │
       ├── Tier 3 Missing Perspective (Pinecone cosine, ~200ms)

✅ Circuit Breaker Protection All LLM calls wrapped in circuit breaker: failure_threshold=3, recovery_timeout=60s. On circuit OPEN: returns None. Callers handle None gracefully — meeting continues uninterrupted. Quorum degrades gracefully; the meeting never stops.

python — consistent hash routing
async def get_prompt(prompt_type: str, org_id: str) -> str:
    """Consistent hashing: same org always gets same A/B bucket."""
    experiment = await db.get_experiment(prompt_type)
    if experiment and experiment.traffic_pct > 0:
        bucket = int(hashlib.md5(org_id.encode()).hexdigest(), 16) % 100
        if bucket < experiment.traffic_pct * 100:
            return experiment.content
    return (await db.get_active_prompt(prompt_type)).content

Non-Negotiable Production Constraints

🔴 These cannot be simplified or skipped Every constraint below exists because violating it causes a production disaster. These are learned from real production systems, not theoretical.
#1Never call Claude without Instructor — raw JSON parsing fails unpredictably at scale; instructor.from_anthropic() always
#2RLS before any production data — no safe way to add PostgreSQL RLS after data exists; do it before first row
#3Anonymization secrets in AWS Secrets Manager ONLY — if these leak, the privacy guarantee is broken for all existing data
#480% test coverage enforced in CI — --cov-fail-under=80; not aspirational; CI fails if below threshold
#5All AI prompts have eval suites — prompts are code; minimum quality scores enforced before promotion
#6No # type: ignore without justification — mypy strict mode; every suppression requires an explanatory comment
#7CELERY_TASK_ALWAYS_EAGER=true in tests — async tasks run synchronously; required for integration test reliability
#8Facilitator brief NEVER visible to participants — separate endpoint, role-checked, not in meeting detail response for participants
#9GET /meetings/{id}/survey/responses returns 404 — this endpoint MUST NOT EXIST; individual responses are never queryable
#10Circuit breaker on all LLM calls — failure_threshold=3, recovery_timeout=60; app degrades, meeting never interrupts
#11All migrations must be backward-compatible — old code must run against new schema; two-phase deploys for column renames
#12No stack traces in production error responses — generic message in production; full detail in development only
#13Never rotate HMAC secrets while survey_open meetings exist — rotating mid-survey breaks duplicate-submission detection
#14All dependencies pinned exactly — security patches within 48hr of CVE; monthly minor upgrades in dedicated PR; no auto-update
#15Never index respondent_hash — indexing it enables efficient "find all responses by this hash" queries; defeats anonymization intent
